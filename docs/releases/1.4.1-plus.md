# LastChat 1.4.1-plus 更新说明

发布日期：2026-03-09

## 本次重点
- 基线对齐到 `54xzh/LastChat` 的 `1.4.0-plus`
- 完整迁移当前聊天界面 UI 重构
- 优化思考 / 工具过程卡的布局、展示策略与滚动性能
- 精简 MCP 工具详情弹层，减少重复信息与阅读负担
- 修复备份恢复中的兼容性提示展示，避免把兼容修正误判为错误

## 主要更新

### 1. 聊天 UI 重构正式并入新基线
- 将当前聊天界面的重构成果迁移到 `1.4.0-plus` 基线上
- 保留新的助手过程卡、思考块、工具调用展示与对齐方案
- 统一用户消息、助手头像、称呼、正文与过程卡之间的视觉关系

### 2. 思考 / 工具过程区体验优化
- 新增更紧凑的思考 / 工具过程展示模式
- 可根据设置在“仅显示当前动作”和“实时追加完整过程”之间切换
- 开始输出正文后，可自动折叠过程区，减少视觉干扰
- 修复审批工具参数显示、描述展示与详情查看中的重复内容问题

### 3. 聊天页性能优化
- 将大型助手回合拆分为更适合 `LazyColumn` 的独立列表项
- 降低过程卡在长内容场景下的滑动卡顿概率
- 减轻思考流式阶段的渲染负担，减少不必要的复杂组件参与
- 精简 MCP 详情弹层内部组件，缓解偶发打开卡顿

### 4. MCP / 工具调用显示优化
- 工具卡摘要更简洁，优先展示 `description`
- 审批场景显示完整可读参数，避免无意义外层 JSON 噪音
- 详情页调整为“调用工具 / 调用结果”双区块结构，便于快速查看关键信息

### 5. 备份恢复兼容性修正
- 备份恢复时的兼容性调整不再被当成错误提示展示
- 减少误导性的“已修复设置问题”提示，使恢复结果更清晰

## 适合关注的回归点
- 长聊天中经过思考 / 工具过程卡时的滚动流畅度
- MCP 工具审批、结果查看与详情弹层展示
- 新旧会话在 `1.4.0-plus` 基线下的显示一致性
- 备份恢复后的提示是否符合预期

## 说明
- 本版本属于基线升级后的整理版本，重点是完成 `1.4.0-plus` 与现有 UI 重构的稳定合并。
- 如需发布到 GitHub Release，可直接将本文件内容作为 Release Body 使用。
